```sql
CREATE OR REPLACE VIEW view_traffic_jam AS
SELECT
	COALESCE(
		avgIf(
			trip_distance / (NULLIF(dateDiff('s', pickup_datetime, dropoff_datetime), 0) / 3600),
			dateDiff('s', pickup_datetime, dropoff_datetime) > 10 AND
			trip_distance > 0.1 AND
			trip_distance < 100
		), 0
	) as avg_speed,
	pickup_hour,
	dictGet('dict_location', 'zone', PU_location_id) as zone,
	max(avg_speed) OVER(PARTITION BY zone) as freeflow_speed,
	(freeflow_speed - avg_speed) / freeflow_speed AS congestion_index
FROM fact_trips
GROUP BY zone, pickup_hour
ORDER BY zone, pickup_hour
```

### VPH (value per hour) and VPM (value per mile)

#### 1. VPH vs VPM по часам
**Инсайт:** Поездка может быть дорогой (высокий VPH), но из-за огромного расстояния она «сжигает» ресурс машины (низкий VPM)
```sql
CREATE OR REPLACE VIEW view_vph_vpm AS
SELECT
	cityHash64(toString(pickup_datetime), toString(net_income)) as trip_id,
	dictGet('dict_taxi_type', 'name', taxi_type_id) as taxi_type,
    ((fare_amount + extra) * 0.75 + tip_amount) AS net_income,
    if(isFinite(net_income / (NULLIF(dateDiff('s', pickup_datetime, dropoff_datetime), 0) / 3600)), 
       net_income / (NULLIF(dateDiff('s', pickup_datetime, dropoff_datetime), 0) / 3600), 0) AS vph,
    if(isFinite(net_income / NULLIF(trip_distance, 0)), 
       net_income / NULLIF(trip_distance, 0), 0) AS vpm,
    pickup_hour,
    pickup_datetime,
    CASE
	    WHEN vph not BETWEEN 5 AND 200 THEN NULL
	    ELSE vph
    END as vph_for_boxplot
FROM fact_trips
WHERE
    fare_amount > 0 
    AND trip_distance > 0 
    AND dateDiff('s', pickup_datetime, dropoff_datetime) > 0
```

#### 2. Earnings stability
Очень важная метрика для понимания риска.

- **Distributions:** `taxi_type` (Yellow vs Green).
- **Metric:** `vph`.
- **Инсайт:** Если «ящик» узкий — заработок стабильный. Если широкий с длинными «усами» — это лотерея: можно заработать $200 в час, а можно $10.

```sql
CREATE OR REPLACE VIEW view_vph AS
SELECT
    cityHash64(toString(pickup_datetime), toString(net_income)) as trip_id,
    taxi_type,
    vph,
    vpm,
    pickup_hour
FROM view_trip_efficiency
WHERE 
    vph BETWEEN 5 AND 200 
    AND vpm BETWEEN 1 AND 100
```

```sql
CREATE OR REPLACE VIEW view_roi AS

```


### NPS (Net Promoter Score)

Мы разделим все поездки на три категории на основе `% чаевых` от стоимости тарифа (`fare_amount`):

1. **Promoters (Промоутеры):** Чаевые **> 20%**. Пассажир в восторге, он готов рекомендовать сервис.
    
2. **Passives (Нейтралы):** Чаевые **от 10% до 20%**. Стандартная вежливость, но без восторга.
    
3. **Detractors (Критики):** Чаевые **< 10%** (включая нулевые). Пассажир недоволен качеством, скоростью или запахом в салоне.
    

**Формула NPS:**

$$NPS = \% \text{Promoters} - \% \text{Detractors}$$
#### 1. Лояльность по времени суток

```sql
CREATE OR REPLACE VIEW view_nps AS
SELECT
    dictGet('dict_taxi_type', 'name', taxi_type_id) as taxi_type,
    pickup_day_of_week,
    pickup_hour,
    (tip_amount / NULLIF(fare_amount, 0)) as tip_ratio,
    CASE 
        WHEN tip_ratio >= 0.20 THEN 100  -- Promoter
        WHEN tip_ratio < 0.10 THEN -100 -- Detractor
        ELSE 0                          -- Passive
    END as nps_score
FROM fact_trips
WHERE fare_amount > 0 
  AND dictGet('dict_payment', 'payment_type', payment_id) = 'Credit card'
```
#### 2. Корреляция лояльности и прибыли
```sql
CREATE OR REPLACE VIEW view_roi_nps_correlation AS
SELECT
    -- Уникальный ключ поездки (как мы делали для Box Plot)
    cityHash64(toString(pickup_datetime), toString(fare_amount)) as trip_id,
    
    -- Метрика NPS (Логика из предыдущего шага)
    (tip_amount / NULLIF(fare_amount, 0)) as tip_ratio,
    CASE 
        WHEN tip_ratio >= 0.20 THEN 100 
        WHEN tip_ratio < 0.10 THEN -100 
        ELSE 0 
    END as nps_score,
    
    -- Метрика ROI (Твоя логика с расходами)
    ((((fare_amount + extra) * 0.75 + tip_amount) - (trip_distance * 0.45)) 
      / NULLIF(trip_distance * 0.45, 0)) * 100 as real_roi,
    
    dictGet('dict_taxi_type', 'name', taxi_type_id) as taxi_type,
    passenger_count,
    trip_distance
FROM fact_trips
WHERE dictGet('dict_payment', 'payment_type', payment_id) = 'Credit card' -- Берем только карты для адекватного NPS
  AND fare_amount > 0 
  AND trip_distance > 0
  AND cityHash64(toString(trip_id)) % 100 < 2
```
